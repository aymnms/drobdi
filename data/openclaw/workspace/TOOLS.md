# TOOLS.md — Notes locales

Les skills définissent _comment_ les outils fonctionnent. Ce fichier est pour _tes_ spécificités — ce qui est propre à ton installation.

## Ce qui va ici

Des choses comme :

- Noms et emplacements des caméras
- Hôtes SSH et alias
- Voix préférées pour la synthèse vocale (TTS)
- Noms des enceintes/pièces
- Surnoms des appareils
- Tout ce qui est spécifique à ton environnement

## Exemples

```markdown
### Caméras

- salon → Zone principale, grand angle 180°
- porte-entrée → Entrée, déclenchement par mouvement

### SSH

- serveur-maison → 192.168.1.100, utilisateur : admin

### TTS

- Voix préférée : "Nova" (chaleureuse, légèrement britannique)
- Enceinte par défaut : HomePod cuisine
```

## Pourquoi séparer ?

Les skills sont partagées. Ton installation t'appartient. Les garder séparées te permet de mettre à jour les skills sans perdre tes notes, et de partager les skills sans exposer ton infrastructure.
