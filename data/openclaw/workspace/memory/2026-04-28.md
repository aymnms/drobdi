- Setup git/SSH durable validé pour OpenClaw dans le container Docker : runtime en user `node`, `sudo` disponible, clé SSH stockée côté host Linux/WSL (pas sur `/mnt/c`) dans un dossier dédié monté sur `/home/node/.ssh`.
- Permissions SSH nécessaires et validées : dossier `.ssh` en `700`, clé privée `id_ed25519` en `600`, clé publique `id_ed25519.pub` en `644`, `known_hosts` en `644`.
- Le test de validation à refaire si besoin est : `ssh -T git@github.com`, puis `git push origin main` depuis `drobsidian`.
- La config git globale fonctionnelle pour le container est minimale : `user.name=drobdi`, pas d’email requis dans ce setup.
- Pour éviter les problèmes de permissions SSH, il faut générer les clés sur le host Linux/WSL et les monter en volume runtime, ne pas les copier dans l’image Docker ni les stocker sur un filesystem Windows monté.
- Contournement XDG retenu pour `gog` : dans le runtime outillé de l’assistant, `XDG_CONFIG_HOME` n’est pas visible par défaut même quand le service/container semble correctement configuré. Il faut donc lancer explicitement les commandes `gog` avec `XDG_CONFIG_HOME=/home/node/.openclaw` ou passer `env.XDG_CONFIG_HOME=/home/node/.openclaw` dans l’outil `exec`.
- Effet attendu du contournement XDG : `gog auth list` et les commandes calendrier retrouvent alors la config OAuth valide dans `/home/node/.openclaw/gogcli/` au lieu de chercher à tort dans `/home/node/.config/gogcli/`.
